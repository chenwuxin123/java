##如何优化项目性能？
```
1.使用缓存
  (1) 数据库缓存
  (2) 页面缓存
  
2.优化数据库连接
  (1) 务必开启Druid连接池的监控统计功能
  (2) 应用程序连接池和数据库最大连接数之间要配套
  (3) 压测时根据错误日志判断哪里需要调参
  
3.优化日志输出：使用Redis、ElasticSearch之类的外部组件

4.程序代码优化
  (1) 前端优化：Tomcat上开启gzip压缩
  (2) 后端优化：例1（String大量循环拼字符串的代码改成用StringBuffer） 例2（SQL索引）
  (3) 数据架构优化：缓存常用稳定数据
  (4) 第三方接口调用优化：
     ·引入缓存，解耦客户端请求和第三方接口调用过程
     ·减少每次的数据量
     ·拉数据改为推数据（可遇不可求）
     
5.数据库设计优化
  (1) 数据库读写分离
  (2) 缓存技术
  (3) 数据库的垂直拆分：商品表和交易表数据量很大，建立在单独的两个数据库里
  (4) 数据库的水平拆分：一张表分成两张表
```
